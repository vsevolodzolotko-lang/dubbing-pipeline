# Інструкція локалізаційного менеджера

Щоденний цикл обробки одного уроку: drop файлу → перевірка → точкове виправлення проблемних сегментів через W_Regen або ElevenLabs Web UI.

---

## Як це працює

Кладеш EN-аудіо у вхідну Drive теку `01_input` → автоматично:
1. Pipeline **архівує попередній урок** у теку `05_archive/{prev_lesson}_{date}/` (зберігає аудіо, VTT і копію всієї таблиці як snapshot)
2. Чистить таби `segments` і `localizations` від старих даних (`voices`/`prompts`/`config` залишаються)
3. Запускає W1 → W2 → W3 на новому файлі
4. У Slack з'являється повідомлення з відсотком `needs_attention` і кнопками: Full audio / Per-segment audio / VTT subtitles / Regen Segments

Усе у фоні.

**Ключовий принцип**: автоматика не дає 100% ідеального результату — частина сегментів потребуватиме перевірки. Два шляхи виправлення:
- **W_Regen** (швидко, через Slack-кнопку) — коли потрібен інший переклад, переписала текст у `text_translated`, хочеш переслухати з новою фразою. Або просто перегенерувати з тими ж параметрами (буває допомагає при випадкових артефактах).
- **ElevenLabs Web UI** (для крайніх випадків) — коли модель раз за разом дає поганий результат через обмеження (надто щільний слот, складна фонетика). Тут ти можеш вручну тюнити темп, додавати паузи `...`, експериментувати з різними прочитаннями.

---

## Структура (де що лежить)

### Google Spreadsheet — 5 вкладок

- **`segments`** — англійський текст уроку, нарізаний на сегменти + переклади на 7 мов. Pipeline заповнює тут поетапно: спочатку EN-текст (після нарізки аудіо), потім колонки `{lang}_text` (після перекладу).
- **`localizations`** — per-segment per-lang результат синтезу: текст, тривалість, швидкість, флаги якості. Заповнюється на стадії синтезу аудіо (фінальна стадія pipeline).
- **`voices`** — голос ElevenLabs для кожної мови + його параметри (`voice_id`, `stability`, `similarity_boost`, `style`, `speed`).
- **`config`** — системні ключі (API keys, IDs тек, налаштування поведінки). Не чіпати без потреби.
- **`prompts`** — промпти для LLM (tone of voice, переклад, редактура). Лінгвіст може коригувати, але обережно — впливає на всі майбутні уроки.

### Google Drive — 5 тек

- **`01_input`** — куди ти кладеш EN-аудіо: [https://drive.google.com/drive/folders/1pucg6dxUB2h_gSyTc655uahPRWidkbA0](https://drive.google.com/drive/folders/1pucg6dxUB2h_gSyTc655uahPRWidkbA0?usp=drive_link).
- **`02_output`** — pipeline складає сюди окремі сегментні WAVи (`{lesson_id}_seg_NNN_{lang}.wav`). Сюди ж кладеш ручкові заміни після виправлення в ElevenLabs UI.
- **`03_full`** — повні `{lesson_id}_full_{lang}.wav` (7 файлів на урок). Те, що йде у фінальну доставку.
- **`04_vtt`** — субтитри (VTT) для кожної мови.
- **`05_archive`** — pipeline сам тукає сюди попередні уроки в підпапки виду `{prev_lesson}_{дата_час}/` (Київський час). Усередині кожної: повна структура 01-04 + копія таблиці (`sheet_snapshot_...`). Якщо потрібно повернути попередній урок — тут усе.

## 1. Запуск нового уроку

**Чистити попередній стан НЕ треба** — pipeline робить це сам:
- Перед стартом нового уроку **archive chain** переміщує файли з `01_input`, `02_output`, `03_full`, `04_vtt` у `05_archive/{попередній_урок}_{дата_час}/`
- Копіює живу таблицю як `sheet_snapshot_...` у ту ж архів-папку
- Витирає рядки 2+ з табів `segments` і `localizations` (header залишається; `voices`/`prompts`/`config` НЕ зачеплено)

Тобто все, що тобі треба:

1. Назви файл як `{lesson_id}.wav` (або `.mp3`) — наприклад `sleep_002.wav` чи `soma_yoga_intro.mp3`. Літери малі, без пробілів, латиниця. Це ім'я стане префіксом усіх сегментів (`sleep_002_seg_001`, ...).
2. Перекинь файл у Drive теку `01_input` (лінк вище).
3. Pipeline запускається автоматично. Прогрес можна спостерігати **в реальному часі**:
   - Спочатку: archive chain (≤ 1 хв) — переміщує файли попереднього уроку в `05_archive`.
   - Далі у sheet `segments` — з'являються рядки з EN-текстом сегментів, потім поступово заповнюються колонки `{lang}_text` по мірі перекладу.
   - Потім у sheet `localizations` — з'являються рядки з результатами синтезу (по 7 рядків на сегмент).
   - Коли всі рядки уроку є у `localizations` — pipeline завершився.
4. У Slack прийде повідомлення `:white_check_mark: Dubbing complete` з:
   - `Needs attention: N% (k / total)` — швидкий індикатор скільки сегментів потребують перевірки.
   - `:file_folder: Full audio (per-lang)` → лінк на `03_full/`.
   - `:musical_note: Per-segment audio` → лінк на `02_output/`.
   - `:closed_book: VTT subtitles` → лінк на `04_vtt/`.
   - `:wrench: Regen Segments` → клік запускає W_Regen для рядків з `needs_retts=TRUE` (без логіну в n8n).

## 2. Перевірка результату

**Найкращий спосіб** — завантажити всі 7 повних `_full_{lang}.wav` у відео/аудіо редактор (CapCut, DaVinci, Reaper, Audition) поряд один з одним. Видно візуально: де треки відрізняються по тривалості, де є дивні паузи чи обриви — це підозрілі місця.

Паралельно дивись колонку `needs_attention` у `localizations` sheet. Має три можливих значення:

| Стан колонки `needs_attention` | Колір (умовне форматування) | Що робити |
|---|---|---|
| Усюди `FALSE` | зелений | Готово. Забирай `_full_{lang}.wav` з теки **full** і відправляй у доставку. |
| 1–3 cells `TRUE` | червоний | Автоматика виявила проблему. Послухай ці сегменти; якщо звучить нормально — постав `FALSE` руками. Якщо обрізано / звучить дивно — виправляй руками (секція 3) або клікни **Regen Segments** у Slack-нотіфаєрі. |
| Cells `REVIEW` | жовтий | `W_Regen` зробив перегенерацію — потрібен ручний прослух. Якщо звучить добре → постав `FALSE`. Якщо все ще погано → постав `TRUE` і повтори правку. |
| Багато `TRUE` (на одній мові або одразу на кількох) | червоний | Поклич automation tech — швидше за все потрібне калібрування або зміна голосу. |

**Тонкий контроль якості (рекомендована практика)**: завантаж `localizations` як CSV і дай LLM (Claude / GPT) з проханням оцінити якість перекладу. Звернути увагу на:
- консистентність звертання на "ти" (informal) у всіх рядках однієї мови
- консистентність гендеру в зверненні (жіночий чи нейтральний — не змішувати)
- false friends і смислові похибки
- природність фраз порівняно з EN оригіналом

Автоматика дає базовий рівень, але людський чек завжди потрібен.

**Якщо знайшла помилки і йдеш виправляти** — на таймлайн у редакторі краще завантажувати окремі сегменти з теки **output** (per-segment WAVи), не повний `_full_*.wav`. Так легше і швидше знайти конкретний сегмент і замінити його після правки.

## 3. Виправлення помилок

Два шляхи, обирай за ситуацією.

### 3a. W_Regen (через Slack-кнопку) — швидкий шлях

Підходить коли:
- Ти переписала `text_translated` у `localizations` (зробила переклад коротшим, додала паузу `...`, виправила гендер) і хочеш переслухати з новим текстом.
- Випадковий артефакт у синтезі — той самий промпт може дати кращий результат другої спроби.

Як:
1. У `localizations` знайди проблемний рядок (фільтруй по `needs_attention=TRUE`).
2. (Опційно) Поправ `text_translated` на що хочеш переслухати. Можна додати `(Опційно) regen_comment` для audit.
3. Постав `needs_retts=TRUE` (колонка стане зеленою — це сигнал що W_Regen підхопить рядок).
4. Натисни **Regen Segments** у Slack-нотіфаєрі (працює без логіну в n8n).
5. Чекай нову Slack-нотифікацію `:arrows_counterclockwise: Regen complete`.
6. Після завершення: у відповідних рядках `needs_attention` стане `REVIEW` (жовтий) — це сигнал що тобі треба послухати. Якщо звучить добре → постав `FALSE`. Якщо все ще погано → постав `TRUE` і повтори (або переходь до 3b).

**Можна одразу декілька рядків і навіть декілька мов** — W_Regen обробляє паралельно (bounded concurrency 5).

### 3b. ElevenLabs Web UI — точковий шлях для крайніх випадків

Підходить коли:
- Pipeline вже зробив 2-3 спроби через W_Regen, і `needs_attention=TRUE` залишається (модель не вкладається у слот).
- Хочеш експериментувати з темпом, паузами, виразністю — W_Regen цього не дасть.

Як:
1. У `voices` подивись параметри голосу мови (`voice_id`, `model`, `stability`, `similarity_boost`, `style`, `speed`).
2. У [ElevenLabs Speech Synthesis](https://elevenlabs.io/app/speech-synthesis) вибери цей голос → встав ті самі параметри → введи текст → згенеруй.
3. Завантаж WAV → одразу підтягуй у редактор замість проблемного сегмента (НЕ треба перекладати назад у Drive — у фінальній доставці ти підмінюєш сегмент сам у редакторі).

**Чому ручний шлях іноді необхідний**: коли автоматика поставила `needs_attention=TRUE` повторно навіть після W_Regen — це сигнал що **щось обмежує правильну генерацію** (надто щільний слот, складна фонетика, межа голосу). W_Regen дасть той самий результат — тільки ручкове втручання (переписати фразу простіше, додати паузу `...`, виставити інший темп) реально розв'язує проблему.

## 4. Голоси

`voices` sheet — це звідки pipeline бере налаштування синтезу. Кожен рядок = одна мова + один голос ElevenLabs.

### Колонки

| Колонка | Що означає |
|---|---|
| `lang` | Код мови (`de`, `es`, `fr`, `it`, `pl`, `pt`, `tr`). Не змінюй для існуючих рядків. |
| `voice_id` | ID голосу з ElevenLabs (копіюється на сторінці голосу в Web UI). |
| `voice_name` | Назва голосу для людей. Інформативно; pipeline не читає. |
| `model` | Зазвичай `eleven_multilingual_v2`. Не міняй якщо не знаєш чому. |
| `stability` | 0–1. Вище = голос стабільніший, нижче = виразніший. Стандарт `0.5`. |
| `similarity_boost` | 0–1. Близькість до клону голосу. Стандарт `0.75`. |
| `style` | 0–1. Стилізація ElevenLabs. `0` = природно. |
| `speed` | Базовий темп голосу. `1.0` = нормальний, нижче — повільніший, вище — швидший. |
| `notes` | Вільне поле для коментарів (наприклад "warmer voice, replaced 2026-06-01"). |

### Як змінити голос для існуючої мови

1. Зайди в [ElevenLabs Voice Library](https://elevenlabs.io/app/voice-library), знайди новий голос → відкрий → скопіюй `voice_id`.
2. У `voices` sheet знайди рядок з потрібною мовою.
3. Встав новий `voice_id` у відповідну комірку. За бажанням онови `voice_name`, `notes`.
4. Послухай у Web UI на коротких прикладах — якщо голос природньо повільніший/швидший за стандарт, підправ `speed`.
5. Запусти наступний урок — pipeline підхопить нові налаштування автоматично.

## 5. Що НЕ чіпати у sheet `localizations`

Системні колонки заповнюються pipeline автоматично. Ручне редагування буде або затерте при наступному W_Regen, або зламає timing-логіку:

- `phase2_outcome`, `shorten_retries_in_synthesize`, `borrowed_sec`, `expansion_attempts`
- `final_speed`, `final_duration_sec`, `real_duration_sec`
- `lead_silence_sec`, `tail_silence_sec`, `slot_start_sec`, `slot_end_sec`, `tts_budget_sec`
- `audio_drive_file_id`
- `last_regen_at` (Київський час, ставить W_Regen)

Колонки які **можна** редагувати:

- `text_translated` — переписати текст перед W_Regen (W_Regen візьме новий варіант) АБО просто для record-keeping після ручного фіксу через ElevenLabs UI.
- `needs_retts` — постав `TRUE` щоб запустити W_Regen на цьому рядку. Pipeline сам поверне у `FALSE` після перегенерації.
- `needs_attention` — manual override. Зніми (`FALSE`) якщо послухала і на слух нормально. Постав (`TRUE`) якщо знайшла проблему що автоматика пропустила. `REVIEW` ставить W_Regen — це сигнал що ти маєш послухати; після прослуху переключай у `FALSE` або `TRUE`.
- `regen_comment` — нотатка для аудиту (наприклад "fixed in EL UI on 2026-06-01: змінила тон").

## 6. Архів попередніх уроків

Кожен запуск W_Master автоматично архівує попередній урок у `05_archive/{prev_lesson}_{YYYY-MM-DD_HH-MM}/`. Усередині кожної архів-папки:

- `01_input/` — оригінальне EN-аудіо попереднього уроку.
- `02_output/` — усі per-segment WAVи.
- `03_full/` — усі повні `_full_{lang}.wav` (7 файлів).
- `04_vtt/` — субтитри.
- `sheet_snapshot_{назва}` — повна копія таблиці на момент архіву (всі 5 табів). Незалежний Google Sheet — редагування живої таблиці після цього моменту НЕ змінюють snapshot.

**Якщо щось пішло не так** (наприклад W2 згенерував жахливий переклад і ти хочеш повернутись до попереднього):
1. Відкрий потрібну архів-папку.
2. Скопіюй файли з підпапок назад у живі `01_input`/`02_output`/`03_full`/`04_vtt`.
3. Відкрий `sheet_snapshot_...` → виділи рядки з табу `segments` (або `localizations`) → скопіюй → встав у живу таблицю.

Архіви не видаляються автоматично — Drive росте з кожним прогоном. Періодично можеш руками видаляти старі архів-папки якщо квота Drive стане проблемою.
