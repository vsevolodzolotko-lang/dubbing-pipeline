# Інструкція локалізаційного менеджера

Щоденний цикл обробки одного уроку: drop файлу → перевірка → ручкове виправлення проблемних сегментів через ElevenLabs Web UI.

---

## Як це працює

Кладеш EN-аудіо у вхідну Drive теку → автоматично запускається pipeline → у Slack з'являється повідомлення з лінком на готовий дубляж 7 мовами (DE / ES / FR / IT / PL / PT / TR). Усе у фоні.

**Ключовий принцип**: автоматика не дає 100% ідеального результату — частина сегментів потребуватиме перевірки та ручкового виправлення. Виправляти найкраще через **ElevenLabs Web UI** з налаштуваннями голосу зі sheet `voices`. Через UI швидше і дає кращий контроль, ніж автоматична перегенерація — особливо коли помилка не випадкова, а наслідок обмежень моделі (тоді автоматика дасть той самий поганий результат).

---

## Структура (де що лежить)

### Google Spreadsheet — 5 вкладок

- **`segments`** — англійський текст уроку, нарізаний на сегменти + переклади на 7 мов. Pipeline заповнює тут поетапно: спочатку EN-текст (після нарізки аудіо), потім колонки `{lang}_text` (після перекладу).
- **`localizations`** — per-segment per-lang результат синтезу: текст, тривалість, швидкість, флаги якості. Заповнюється на стадії синтезу аудіо (фінальна стадія pipeline).
- **`voices`** — голос ElevenLabs для кожної мови + його параметри (`voice_id`, `stability`, `similarity_boost`, `style`, `speed`).
- **`config`** — системні ключі (API keys, IDs тек, налаштування поведінки). Не чіпати без потреби.
- **`prompts`** — промпти для LLM (tone of voice, переклад, редактура). Лінгвіст може коригувати, але обережно — впливає на всі майбутні уроки.

### Google Drive — 4 теки

- **input** — куди ти кладеш EN-аудіо: [https://drive.google.com/drive/folders/1pucg6dxUB2h_gSyTc655uahPRWidkbA0](https://drive.google.com/drive/folders/1pucg6dxUB2h_gSyTc655uahPRWidkbA0?usp=drive_link).
- **output** — pipeline складає сюди окремі сегментні WAVи (`{lesson_id}_seg_NNN_{lang}.wav`). Сюди ж кладеш ручкові заміни після виправлення в ElevenLabs UI.
- **full** — повні `{lesson_id}_full_{lang}.wav` (7 файлів на урок). Те, що йде у фінальну доставку.
- **vtt** — субтитри (VTT) для кожної мови.

## 1. Запуск нового уроку

1. Назви файл як `{lesson_id}.wav` — наприклад `sleep_002.wav` чи `soma_yoga_intro.wav`. Літери малі, без пробілів, латиниця. Це ім'я стане префіксом усіх сегментів (`sleep_002_seg_001`, ...).
2. Перекинь файл у Drive теку **input** (лінк вище).
3. Pipeline запускається автоматично. Прогрес можна спостерігати **в реальному часі**:
   - Спочатку у sheet `segments` — з'являються рядки з EN-текстом сегментів, потім поступово заповнюються колонки `{lang}_text` по мірі перекладу.
   - Потім у sheet `localizations` — з'являються рядки з результатами синтезу (по 7 рядків на сегмент).
   - Коли всі рядки уроку є у `localizations` — pipeline завершився.
4. У Slack прийде повідомлення `:white_check_mark: Dubbing complete` з лінком на Drive теку **full**.

## 2. Перевірка результату

**Найкращий спосіб** — завантажити всі 7 повних `_full_{lang}.wav` у відео/аудіо редактор (CapCut, DaVinci, Reaper, Audition) поряд один з одним. Видно візуально: де треки відрізняються по тривалості, де є дивні паузи чи обриви — це підозрілі місця.

Паралельно дивись колонку `needs_attention` у `localizations` sheet:

| Стан колонки `needs_attention` | Що робити |
|---|---|
| Усюди `FALSE` | Готово. Забирай `_full_{lang}.wav` з теки **full** і відправляй у доставку. |
| 1–3 cells `TRUE` | Послухай ці сегменти; якщо звучить нормально — лиши. Якщо обрізано / звучить дивно — виправляй руками (секція 3). |
| Багато `TRUE` (на одній мові або одразу на кількох) | Поклич automation tech — швидше за все потрібне калібрування або зміна голосу. |

**Тонкий контроль якості (рекомендована практика)**: завантаж `localizations` як CSV і дай LLM (Claude / GPT) з проханням оцінити якість перекладу. Звернути увагу на:
- консистентність звертання на "ти" (informal) у всіх рядках однієї мови
- консистентність гендеру в зверненні (жіночий чи нейтральний — не змішувати)
- false friends і смислові похибки
- природність фраз порівняно з EN оригіналом

Автоматика дає базовий рівень, але людський чек завжди потрібен.

**Якщо знайшла помилки і йдеш виправляти** — на таймлайн у редакторі краще завантажувати окремі сегменти з теки **output** (per-segment WAVи), не повний `_full_*.wav`. Так легше і швидше знайти конкретний сегмент і замінити його після правки.

## 3. Виправлення помилок руками (ElevenLabs Web UI)

Якщо знайшла проблемний сегмент, найшвидший шлях — згенерувати заміну через [ElevenLabs Web UI](https://elevenlabs.io/app/speech-synthesis) з тими самими параметрами голосу, що використовує pipeline (їх видно у `voices`: `voice_id`, `model`, `stability`, `similarity_boost`, `style`, `speed`).

Згенерований файл збережи з тим самим іменем (`{lesson_id}_seg_NNN_{lang}.wav`) і поклади на місце у Drive теку **output**, перезаписавши старий. У редакторі підтягуй вже виправлений сегмент і збирай фінальний трек.

**Чому ручний шлях кращий за автоматичну перегенерацію**: коли автоматика поставила `needs_attention=TRUE`, це часто означає що **щось обмежує правильну генерацію** (надто щільний слот, складна фраза, межа голосу). Повторний автоматичний прогін дасть той самий результат — тільки ручкове втручання (переписати фразу простіше, додати паузу `...`, виставити інший темп) реально розв'язує проблему.

## 4. Голоси

`voices` sheet — це звідки pipeline бере налаштування синтезу. Кожен рядок = одна мова + один голос ElevenLabs.

### Колонки

| Колонка | Що означає |
|---|---|
| `lang` | Код мови (`de`, `es`, `fr`, `it`, `pl`, `pt`, `tr`). Не змінюй для існуючих рядків. |
| `voice_id` | ID голосу з ElevenLabs (копіюється на сторінці голосу в Web UI). |
| `voice_name` | Назва голосу для людей. Інформативно; pipeline не читає. |
| `model` | Зазвичай `eleven_multilingual_v2`. Не міняй якщо не знаєш чому. |
| `stability` | 0–1. Вище = голос стабільніший, нижче = виразніший. Стандарт `0.5`. |
| `similarity_boost` | 0–1. Близькість до клону голосу. Стандарт `0.75`. |
| `style` | 0–1. Стилізація ElevenLabs. `0` = природно. |
| `speed` | Базовий темп голосу. `1.0` = нормальний, нижче — повільніший, вище — швидший. |
| `notes` | Вільне поле для коментарів (наприклад "warmer voice, replaced 2026-06-01"). |

### Як змінити голос для існуючої мови

1. Зайди в [ElevenLabs Voice Library](https://elevenlabs.io/app/voice-library), знайди новий голос → відкрий → скопіюй `voice_id`.
2. У `voices` sheet знайди рядок з потрібною мовою.
3. Встав новий `voice_id` у відповідну комірку. За бажанням онови `voice_name`, `notes`.
4. Послухай у Web UI на коротких прикладах — якщо голос природньо повільніший/швидший за стандарт, підправ `speed`.
5. Запусти наступний урок — pipeline підхопить нові налаштування автоматично.

---

## 5. Що НЕ чіпати у sheet `localizations`

Системні колонки заповнюються pipeline автоматично. Ручне редагування буде або затерте при наступному запуску, або зламає timing-логіку:

- `phase2_outcome`, `shorten_retries_in_synthesize`, `borrowed_sec`, `expansion_attempts`
- `final_speed`, `final_duration_sec`, `real_duration_sec`
- `lead_silence_sec`, `tail_silence_sec`, `slot_start_sec`, `slot_end_sec`, `tts_budget_sec`
- `audio_drive_file_id`
- `last_regen_at`

Колонки які **можна** редагувати:

- `text_translated` — виправляти текст (для record-keeping після ручного фіксу через ElevenLabs UI)
- `needs_attention` — manual override (знімай флаг якщо послухала і на слух нормально, або став якщо знайшла проблему що автоматика пропустила)
- `regen_comment` — нотатка для аудиту (наприклад "fixed in EL UI on 2026-06-01: змінила тон")
