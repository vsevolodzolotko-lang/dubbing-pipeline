# Інструкція локалізаційного менеджера

Щоденний цикл обробки одного уроку: drop файлу → перевірка → ручкове виправлення проблемних сегментів через ElevenLabs Web UI. Технічні деталі — у решті `docs/` (посилання в кінці).

---

## Як це працює

Кладеш EN-аудіо у вхідну Drive теку → автоматично запускається pipeline → у Slack з'являється повідомлення з лінком на готовий дубляж 7 мовами (DE / ES / FR / IT / PL / PT / TR). Усе у фоні.

**Ключовий принцип**: автоматика не дає 100% ідеального результату — частина сегментів потребуватиме перевірки та ручкового виправлення. Виправляти найкраще через **ElevenLabs Web UI** з налаштуваннями голосу зі sheet `voices`. Робота через UI швидша і дає кращий контроль, ніж автоматична перегенерація — особливо коли помилка не випадкова, а наслідок обмежень моделі (тоді автоматика дасть той самий поганий результат).

---

## 1. Запуск нового уроку

1. Назви файл як `{lesson_id}.wav` — наприклад `sleep_002.wav` чи `soma_yoga_intro.wav`. Літери малі, без пробілів, латиниця. Це ім'я стане префіксом усіх сегментів (`sleep_002_seg_001`, ...).
2. Перекинь файл у вхідну Drive теку: [https://drive.google.com/drive/folders/1pucg6dxUB2h_gSyTc655uahPRWidkbA0](https://drive.google.com/drive/folders/1pucg6dxUB2h_gSyTc655uahPRWidkbA0?usp=drive_link).
3. Pipeline запускається автоматично. Прогрес можна спостерігати **в реальному часі** у sheet `localizations` — рядки заповнюються в міру обробки (translate → synthesize). Коли всі рядки уроку з'явились — pipeline завершився.
4. У Slack прийде повідомлення `:white_check_mark: Dubbing complete` з лінком на Drive теку із 7 файлами `{lesson_id}_full_{lang}.wav`.

---

## 2. Перевірка результату

**Найкращий спосіб** — завантажити всі 7 повних `_full_{lang}.wav` у відео/аудіо редактор (CapCut, DaVinci, Reaper, Audition) поряд один з одним. Видно візуально: де треки відрізняються по тривалості, де є дивні паузи чи обриви — це підозрілі місця.

Паралельно дивись колонку `needs_attention` у `localizations` sheet:

| Стан колонки `needs_attention` | Що робити |
|---|---|
| Усюди `FALSE` | Готово. Забирай `_full_{lang}.wav` з Drive і відправляй у доставку. |
| 1–3 cells `TRUE` | Послухай ці сегменти; якщо звучить нормально — лиши. Якщо обрізано / звучить дивно — виправляй руками (секція 3). |
| Багато `TRUE` (на одній мові або одразу на кількох) | Поклич automation tech — швидше за все потрібне калібрування або зміна голосу. |

**Тонкий контроль якості (рекомендована практика)**: завантаж `localizations` як CSV і дай LLM (Claude / GPT) з проханням оцінити якість перекладу. Звернути увагу на:
- консистентність звертання на "ти" (informal) у всіх рядках однієї мови
- консистентність гендеру в зверненні (жіночий чи нейтральний — не змішувати)
- false friends і смислові похибки
- природність фраз порівняно з EN оригіналом

Автоматика дає базовий рівень, але людський чек завжди потрібен.

**Якщо знайшла помилки і йдеш виправляти** — на таймлайн у редакторі краще завантажувати окремі сегменти з теки `output` (per-segment WAVs), не повний `_full_*.wav`. Так легше і швидше знайти конкретний сегмент і замінити його після правки.

## 3. Виправлення помилок руками (ElevenLabs Web UI)

Коли якийсь сегмент звучить погано:

1. У `localizations` sheet знайди рядок (`segment_id` + `lang`) і відкрий поряд sheet `voices` — знайди рядок з потрібною мовою.
2. У `voices` подивись параметри для цієї мови: `voice_id`, `model`, `stability`, `similarity_boost`, `style`, `speed`. Це ті самі налаштування, з якими автоматика синтезує.
3. Відкрий [ElevenLabs Web UI](https://elevenlabs.io/app/speech-synthesis), вибери цей голос (за `voice_id`), виставь точно ті самі параметри з `voices`.
4. Встав виправлений текст (відредагуй у `text_translated` у sheet або просто скопіюй і виправ на льоту в UI).
5. Згенеруй. Послухай. Якщо ок — завантаж WAV.
6. Замінь файл у Drive теці `output` — назва має лишитись та сама (`{lesson_id}_seg_NNN_{lang}.wav`).
7. У редакторі підтягуй вже виправлений сегмент і збирай фінальний трек.

**Чому ручний шлях кращий за автоматичну перегенерацію**: якщо автоматика поставила `needs_attention=TRUE`, це часто означає що **щось обмежує правильну генерацію** (надто щільний слот, складна фраза, межа голосу). Повторний автоматичний прогін дасть той самий результат — тільки ручкове втручання (переписати фразу простіше, змінити темп у UI, додати паузу `...`) реально розв'язує проблему.

Якщо помилок мало (1–5) — виправити швидше руками за 5–10 хв. Якщо помилок багато — клич automation tech, можливо CPS-оцінка або голос потребує калібрування.

## 4. Голоси: як змінювати і додавати

`voices` sheet — single source of truth для синтезу. Кожен рядок = одна мова + один голос ElevenLabs.

### Колонки

| Колонка | Що означає |
|---|---|
| `lang` | Код мови (`de`, `es`, `fr`, `it`, `pl`, `pt`, `tr`). Не змінюй для існуючих рядків. |
| `voice_id` | ID голосу з ElevenLabs (можна скопіювати на сторінці голосу в Web UI). |
| `voice_name` | Назва голосу для людей. Інформативно; pipeline не читає. |
| `model` | Зазвичай `eleven_multilingual_v2`. Не міняй якщо не знаєш чому. |
| `stability` | 0–1. Вище = голос стабільніший, нижче = виразніший. Стандарт `0.5`. |
| `similarity_boost` | 0–1. Близькість до клону голосу. Стандарт `0.75`. |
| `style` | 0–1. Стилізація ElevenLabs. `0` = природно. |
| `speed` | Базовий темп голосу. `1.0` = нормальний, `0.86` повільніший (для медитативних голосів), `1.1` швидший. |
| `notes` | Вільне поле для коментарів (наприклад "warmer voice, replaced 2026-06-01"). |

### Як змінити голос для існуючої мови

1. Зайди в [ElevenLabs Voice Library](https://elevenlabs.io/app/voice-library), знайди новий голос → відкрий → скопіюй `voice_id`.
2. Зайди у `voices` sheet, знайди рядок з потрібною мовою.
3. Встав новий `voice_id` у відповідну комірку. За бажанням онови `voice_name`, `notes`.
4. Послухай у Web UI на коротких прикладах — якщо голос природньо повільніший/швидший за стандарт, підправ `speed` (наприклад `0.86` для дуже спокійного медитативного голосу).
5. Запусти наступний урок — pipeline підхопить нові налаштування автоматично.

### Як додати нову мову

1. Додай новий рядок у `voices`: код мови + `voice_id` + налаштування (стандартні параметри як у решти рядків).
2. У `config` sheet знайди ключ `active_langs` (наприклад `de,es,fr,it,pl,pt,tr`) → додай новий код через кому.
3. У `config` додай ключ `cps_estimate_{lang}` зі стартовим значенням `12` (потім калібрувати після 2–3 уроків).
4. Перевір що `prompts/translate_system` промпт підтримує цю мову (правила informal/formal, false friends — поглянь у [docs/localization_rules.md](localization_rules.md)).
5. Прогни тестовий урок, перевір якість, при потребі калібруй (див. [docs/cps_calibration.md](cps_calibration.md)).

### Що дає налаштування `speed`

`speed` — це базова швидкість, з якою pipeline просить ElevenLabs озвучити текст. Її варто підправити коли:
- голос природньо швидкий і слова "проковтуються" → опусти до `0.9` або `0.86`
- голос природньо повільний і не вкладається у слоти → підніми до `1.05`

Не міняй `speed` різко за один раз — крок `0.05` за раз, послухай результат, потім далі.

---

## 5. Що НЕ чіпати у sheet `localizations`

Системні колонки заповнюються pipeline автоматично. Ручне редагування буде або затерте при наступному запуску, або зламає timing-логіку:

- `phase2_outcome`, `shorten_retries_in_synthesize`, `borrowed_sec`, `expansion_attempts`
- `final_speed`, `final_duration_sec`, `real_duration_sec`
- `lead_silence_sec`, `tail_silence_sec`, `slot_start_sec`, `slot_end_sec`, `tts_budget_sec`
- `audio_drive_file_id`
- `last_regen_at`

Колонки які **можна** редагувати:

- `text_translated` — виправляти текст (для record-keeping після ручного фіксу через ElevenLabs UI)
- `needs_attention` — manual override (знімай флаг якщо послухала і на слух нормально, або став якщо знайшла проблему що автоматика пропустила)
- `regen_comment` — нотатка для аудиту (наприклад "fixed in EL UI on 2026-06-01: змінила тон")
